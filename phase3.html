Interface graphique

Mise en page

3 points code la mise en page respecte l'image suivante (cliquez pour élargir)

Liste

2 points code la liste gère correctement les augmentations et diminutions de quantité ainsi que la suppression quand la quantité tombe à 0.

1 point code le montant à payer est correctement maintenu à jour (à côté du bouton SCANNER).

Comportement

  • Le scan d'un code barre via l'utilisation d'un Intent (qui démarre la caméra de la tablette) provoque :
    • 1 point code l'ajout à la liste du produit s'il existe dans l'inventaire.
    • 1 point code l'ouverture du dialogue d'ajout à l'inventaire si le code barre n'existe pas. (voir image à droite)
  • 1 point code Le paiement provoque :
    • la sauvegarde de l'achat de cette liste (voir persistance plus bas)
    • l'affichage de tous les achats (montant + date + liste des produits avec quantité) depuis l'installation de l'application dans le log de la tablette avec le tag "ACHATS"
    • l'effacement de la liste d'achats courante (on peut alors commencer un nouvel achat)

Menu de l'activité

  • 1 point code Créer inventaire doit remplir l'inventaire avec 9 produits avec des codes à barre valides. Les produits doivent correspondre à votre feuille de codes barre (voir plus bas)
  • 1 point code Créer une liste doit créer une liste d'achats en prenant des produits au hasard avec des quantités aléatoires entre 1 et 5
  • 1 point code Vider inventaire doit vider l'inventaire des produits existants

Base de données

Produit

1 point code 3 points test Vous devez implanter la persistance d'un produit avec les champs appropriés. Votre code doit être validé par des tests automatisés.

Vous devez implanter les opérations dans la couche de service suivantes au minimum (vous pouvez ajouter des opérations pour faciliter les tests ou autre).

Pour chaque méthode, vous devez tester les cas normaux et les cas exceptionnels (code barre absent, mal formé etc.)

  • création d'une question
  • création d'un vote
  • récupération des votes pour une question donnée
  • suppression de tous les produits

Achat

1 point code 2 points test Vous devez implanter la persistance d'un achat. Les opérations obligatoires sont:

  • création d'un achat
  • récupération de l'ensemble des achats
  • autre cas ...

Fonctionnement global

4 points négatifs Votre application doit fonctionner sans plantage et correctement. Ce pointage fonctionne en négatif. Si l'application fonctionne correctement en tout temps, vous conservez votre note. Dans le cas contraire, vous perdez des points avec un maximum de 5.

  • plantage de l'application -1 point
  • interface illisible -1 point
  • information affichée incohérente -1 point
  • autre cas ...

Feuille de code barre

1 point négatif Vous devez joindre à votre remise une feuille avec la liste des 9 produits de votre inventaire avec une ligne par produit.

  • image du code barre
  • nom du produit
  • prix unitaire

Modèle ici

Vous devez également inclure une ligne vierge avec un code barre valide qui n'existe pas dans l'inventaire. L'absence de cette feuille entraîne la perte d'un point.
