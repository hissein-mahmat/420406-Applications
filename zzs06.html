Explication de l' enum en Java. Ne pas utiliser un enum comme stockage.

Principes pour écrire des tests.

  • Toujours pouvoir exécuter ce que j'écris, tests au fur et à mesure
  • Une classe de code = une classe de test
  • Une méthode = plusieurs tests
    • Quand ça fonctionne
    • Quand ça exceptionne
    • Quand ça boucle plante etc.
  • Comment utiliser la couverture de code

Autres éléments concernant les tests

  • Quoi tester? Est-ce qu'on teste tout? Ce qui est risqué, nouveau, cher etc.
  • Quand doit-on tester? Quand est-ce qu'on écrit les tests? Quand est-ce qu'on les exécute?
  • Hasard dans les tests? Exemple avec Fibonacci et test de plus le nombre est grand plus son fibo est grand.
