Travail 2 Questionnaire et tests
  • Accueil

Objectif

Ce travail consiste à programmer un service pour gérer des questions, le calcul des votes ainsi que la compilation de statistiques.

Vous devrez programmer les éléments demandés mais également mettre en place des tests automatiques afin de démontrer la qualité de votre code.

Vous devrez définir des classes d'exception significatives pour les problèmes possibles. Vous les regrouperez dans le package exceptions (1 point).

Fichiers de départ

Vous trouverez une interface décrivant les méthodes à implémenter ici. Vous pouvez télécharger l'ensemble du code du cours ici

Nom du package et des classes

1 point Vous devez avoir:

  • un package appelé org.votrenom qui regroupe tous les packages suivants :
    • un package interfaces pour la classe d'interface fournie
    • un package impl pour vos classes d'implémentation
    • un package modele pour vos classes du modèle
    • un package exceptions pour vos classes d'exceptions

Modèles

Constructeur

Vos constructeurs doivent avoir comme argument tous les paramètres de votre classe.

Implémentation du Service

Ajouter une question

Votre service doit permettre d'ajouter une question.

  • 1 point code 2 points test Pour simuler le fonctionnement de la BD, le id de la question est attribué une fois les vérifications effectuées, au moment d'ajouter la question dans votre structure de données. La méthode doit refuser les objets qui ont un id non nul. Si la méthode s'est exécutée sans erreur, l'objet passé en paramètre doit avoir un id fixé. Si on ajoute 2 questions, les 2 questions devraient recevoir des ids différents.
  • 1 point code 2 points test La longueur de la question doit être entre 5 (inclus) et 255 (inclus) caractères.
  • 2 point code 3 points test La question doit être unique. (Note : bbb est considéré identique à BBB)

  • Une question peut avoir plusieurs votes.
Ajouter un vote

Votre service doit permettre d'ajouter un vote.

  • 1 point code 2 points test Pour simuler le fonctionnement de la BD, le id du vote est attribué une fois les vérifications effectuées, au moment d'ajouter la question dans votre structure de données.
  • 1 point code 2 points test Les votes sont un indice de satisfaction de 0 à 5 (inclus).
  • 1 point code 2 points test Une même personne ne peut pas voter deux fois pour la même question. (Note : Maude est considéré identique à maude)
  • 1 point code 2 points test Un vote doit être associé à une question. (Attention : dans la suite, la base de données est de type SQL, prendre ça en compte pour modéliser les données)
Obtenir les questions triées

2 point code 3 points test Votre service doit retourner une version triée de la liste des questions en ordre décroissant du nombre de votes et, si le nombre de votes est identique, en ordre croissant alphabétique.

Note : L'ordre alphabétique doit mettre J (j majuscule) avant k (k minuscule).

Calcul de la distribution

1 point code 2 points test Votre service doit calculer la distribution de toutes les valeurs des votes d'une question. Par exemple, si deux usagers ont voté 2/5 et un usager a voté 5/5, la distribution sera de 2 pour 2/5, 1 pour 5/5 et 0 pour toutes les autres valeurs.

Calcul de la moyenne

1 point code 2 points test Votre service doit calculer la moyenne de toutes les valeurs des votes d'une question. Par exemple, si deux usagers ont voté 2/5 et un usager a voté 5/5, la moyenne sera 3/5 (donc 3).

Calcul de l'écart-type

1 point code 2 points test Votre service doit calculer l'écart-type de toutes les valeurs des votes d'une question. Par exemple, si deux usagers ont voté 2/5 et un usager a voté 5/5, l'écart-type sera 1.4142135623731...

Couverture de code

4 points test Regroupez tout le code que vous avez produit dans le package impl pour faciliter le calcul de la couverture de code (pourcentage des lignes de code).

  • 1 point entre 50% et 75%
  • 2 points entre 75% et 85%
  • 3 points entre 85% et 95%
  • 4 points > 95%

Manipulation par les interfaces

4 points négatifs Tous vos traitements doivent utiliser les méthodes définies dans les interfaces. Vous devez manipuler les objets par l'interface fournie plutôt que par vos classes d'implémentations. Vous perdrez un point à chaque fois que :

  • Vous accédez directement à un champ d'une classe depuis une autre classe (dans le code ou dans les tests)
  • Vous utilisez une méthode définie dans vos classes mais qui n'existe pas dans l'interface (dans le code ou dans les tests)
